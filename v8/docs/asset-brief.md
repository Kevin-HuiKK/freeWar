# V8 Asset Brief

## Visual Direction

Use the handwritten design sheet as the visual seed, but make the final game screen cleaner and readable.

Target feeling:

- Hand-drawn grand strategy map.
- Aged notebook paper, ink outlines, red-pencil route planning.
- Organic island coastlines.
- Clear city nodes and connection lines.
- UI feels like a map command desk, not a cartoon board.

Avoid:

- Square tile maps.
- Plastic mobile-game buttons.
- Overly realistic satellite maps.
- Dark fantasy textures that hide roads and city nodes.
- Dense decorative labels that compete with gameplay icons.

## Main Background

### `assets/backgrounds/world_map_base.png`

Purpose: fixed V8 world map background.

Size:

- Desktop target: 1920x1080.
- Safe gameplay area: central 1680x920.
- Leave the right 300px visually calmer for UI panels.

Content:

- 3 large landmasses.
- 4-7 smaller islands.
- Several natural sea channels.
- Coastlines drawn with ink.
- Land interiors have subtle hills, forests, and rivers.
- No city nodes, no labels, no roads, no borders.

Composition:

- Main continent in lower-left and center.
- Dense central landmass for early conflict.
- Right-side island group for naval expansion.
- Top-left isolated island for late-game sea route value.

## Routes

Routes can be drawn dynamically in canvas/SVG, but texture overlays are useful.

Visual rules:

- Normal road: dark brown hand-drawn line.
- Trade route: dark line with small gold moving dots.
- Military road: stronger line with red edge.
- Broken route: dashed black/red line with gap marks.
- Sea lane: blue-gray curved dashed line.
- Blockaded sea lane: sea lane with red cross marks.

Implementation preference:

- Draw routes in code for flexibility.
- Use small repeating texture stamps only if dynamic drawing looks too clean.

## Borders

Borders should be generated by code using city influence. Use transparent overlays or canvas strokes.

Visual rules:

- Player: warm gold translucent fill, pale border.
- Enemy red: red translucent fill.
- Enemy blue: blue translucent fill.
- Neutral: low-opacity gray.
- Contested: striped or pulsing border.

## UI Assets

UI should be mostly CSS/canvas, with a few reusable raster assets if desired.

Required panels:

- City detail panel.
- Route detail panel.
- Top resource strip.
- Bottom action bar.
- Minimap frame.
- Tooltip frame.

Style:

- Dark ink panels with parchment highlights.
- 4px or smaller border radius.
- No oversized rounded pill buttons.
- Icon-first controls for common actions.

## Animation Notes

Use lightweight animation:

- Trade dots move along route paths.
- City growth ring slowly fills.
- Broken route sparks briefly when cut.
- Siege city border pulses red.
- Port sea lane has slow flowing dash offset.

Do not animate the background map.
